Description: Use the chrome.bluetooth module to connect to a Bluetooth device. For example, the zephyr_hxm sample uses the Bluetooth API to fetch heart rate data from a Zephyr HXM device (a very simple heart rate monitor driver).
Availability: Google Chrome Dev Channel Only
Permissions: "bluetooth"
Learn more: Accessing Hardware Devices

API 参考：chrome.bluetooth

类型

AdapterState

( object )

AdapterState 的属性

address ( string )
The address of the adapter, in the format 'XX:XX:XX:XX:XX:XX'.
name ( string )
The human-readable name of the adapter.
powered ( boolean )
Indicates whether or not the adapter has power.
available ( boolean )
Indicates whether or not the adapter is available (i.e. enabled).
discovering ( boolean )
Indicates whether or not the adapter is currently discovering.

Device

( object )

Device 的属性

address ( string )
The address of the device, in the format 'XX:XX:XX:XX:XX:XX'.
name ( string )
The human-readable name of the device.
paired ( boolean )
Indicates whether or not the device is paired with the system.
bonded ( boolean )
Indicates whether or not the device is bonded with the system. A device is bonded if it is paired and high-security link keys have been exchanged so that connections may be encrypted.
connected ( boolean )
Indicates whether the device is currently connected to the system.

ServiceRecord

( object )

ServiceRecord 的属性

name ( string )
The name of the service.
uuid ( optional string )
The UUID of the service.

Socket

( object )

Socket 的属性

device ( Device )
The remote Bluetooth device associated with this socket.
serviceUuid ( string )
The remote Bluetooth service associated with this socket.
id ( integer )
An identifier for this socket that should be used with the read/write/disconnect methods.

OutOfBandPairingData

( object )

OutOfBandPairingData 的属性

hash ( arraybuffer )
Simple Pairing Hash C. Always 16 octets long.
randomizer ( arraybuffer )
Simple Pairing Randomizer R. Always 16 octets long.

GetDevicesOptions

( object )

GetDevicesOptions 的属性

uuid ( optional string )
Only devices providing a service with a UUID that matches |uuid| will be returned.
name ( optional string )
Only devices providing a service with a name that matches |name| will be returned.
deviceCallback ( function )
Called for each matching device. Note that a service discovery request must be made to each non-matching device before it can be definitively excluded. This can take some time.
参数
device ( Device )

GetServicesOptions

( object )

GetServicesOptions 的属性

deviceAddress ( string )
The address of the device to inquire about. |deviceAddress| should be in the format 'XX:XX:XX:XX:XX:XX'.

ConnectOptions

( object )

ConnectOptions 的属性

deviceAddress ( string )
The connection is made to the device at |deviceAddress|. |deviceAddress| should be in the format 'XX:XX:XX:XX:XX:XX'.
serviceUuid ( string )
The connection is made to the service with UUID |serviceUuid|.

DisconnectOptions

( object )

DisconnectOptions 的属性

socketId ( integer )
The socket to disconnect.

ReadOptions

( object )

ReadOptions 的属性

socketId ( integer )
The socket to read from.

WriteOptions

( object )

WriteOptions 的属性

socketId ( integer )
The socket to write to.
data ( arraybuffer )
The data to write.

SetOutOfBandPairingDataOptions

( object )

SetOutOfBandPairingDataOptions 的属性

address ( string )
The address of the remote device that the data should be associated with. |deviceAddress| should be in the format 'XX:XX:XX:XX:XX:XX'.
data ( optional OutOfBandPairingData )
The Out Of Band Pairing Data. If this is omitted, the data for the device is cleared instead.

StartDiscoveryOptions

( object )

StartDiscoveryOptions 的属性

deviceCallback ( function )
Called for each device that is discovered.
参数
device ( Device )

方法

getAdapterState

chrome.bluetooth.getAdapterState(function callback)

Get information about the Bluetooth adapter.

参数

callback ( function )
Called with an AdapterState object describing the adapter state.

回调函数

callback 参数应该指定一个如下形式的函数： 

function(AdapterState result) {...};
result ( AdapterState )

getDevices

chrome.bluetooth.getDevices(GetDevicesOptions options, function callback)

Get a bluetooth devices known to the system. Known devices are either currently bonded, or have been bonded in the past.

参数

options ( GetDevicesOptions )
Controls which devices are returned and provides |deviceCallback|, which is called for each matching device.
callback ( function )
Called when the search is completed. |options.deviceCallback| will not be called after |callback| has been called.

回调函数

callback 参数应该指定一个如下形式的函数： 

function() {...};

getServices

chrome.bluetooth.getServices(GetServicesOptions options, function callback)

Get a list of services provided by a device.

参数

options ( GetServicesOptions )
callback ( function )

回调函数

callback 参数应该指定一个如下形式的函数： 

function(array of ServiceRecord result) {...};
result ( array of ServiceRecord )

connect

chrome.bluetooth.connect(ConnectOptions options, function callback)

Connect to a service on a device.

参数

options ( ConnectOptions )
The options for the connection.
callback ( function )
Called when the connection is established with a Socket that can be used to communicate with |device|.

回调函数

callback 参数应该指定一个如下形式的函数： 

function(Socket result) {...};
result ( Socket )

startDiscovery

chrome.bluetooth.startDiscovery(StartDiscoveryOptions options, function callback)

Start discovery. Discovered devices will be returned via the |onDeviceDiscovered| callback. Discovery will fail to start if it is already in progress. Discovery can be resource intensive: stopDiscovery should be called as soon as possible.

参数

The options for this function.
callback ( optional function )
Called to indicate success or failure.
