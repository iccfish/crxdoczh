Description: Use the chrome.app.window module to create windows. Windows can either be shells or panels. Shell windows have no browser chrome. Panel windows are the same as shell windows, except they have different size and position restrictions, for example, a chat panel.
Availability: Google Chrome 23
Permissions: None

CreateWindowOptions

( object )

CreateWindowOptions 的属性

id ( optional string )
Id to identify the window. This will be used to remember the size and position of the window and restore that geometry when a window with the same id (and no explicit size or position) is later opened.
minWidth ( optional integer )
Minimum width of the window.
minHeight ( optional integer )
Minimum height of the window.
maxWidth ( optional integer )
Maximum width of the window.
maxHeight ( optional integer )
Maximum height of the window.
frame ( optional string )
Frame type: 'none' or 'chrome' (defaults to 'chrome').
bounds ( optional Bounds )
Size of the content in the window (excluding the titlebar). If specified in addition to any of the left/top/width/height parameters, this field takes precedence. If a frameBounds is specified, the frameBounds take precedence.
transparentBackground ( optional boolean )
Enable window background transparency. Only supported in ash. Requires experimental API permission.
hidden ( optional boolean )
If true, the window will be created in a hidden state. Call show() on the window to show it once it has been created. Defaults to false.
singleton ( optional boolean )
By default if you specify an id for the window, the window will only be created if another window with the same id doesn't already exist. If a window with the same id already exists that window is activated instead. If you do want to create multiple windows with the same id, you can set this property to false.

方法

create

chrome.app.window.create(string url, CreateWindowOptions options, function callback)

The size and position of a window can be specified in a number of different ways. The most simple option is not specifying anything at all, in which case a default size and platform dependent position will be used.

Another option is to use the top/left and width/height properties, which will always put the window at the specified coordinates with the specified size.

Yet another option is to give the window a (unique) id. This id is then used to remember the size and position of the window whenever it is moved or resized. This size and position is then used instead of the specified bounds on subsequent opening of a window with the same id. If you need to open a window with an id at a location other than the remembered default, you can create it hidden, move it to the desired location, then show it.

You can also combine these various options, explicitly specifying for example the size while having the position be remembered or other combinations like that. Size and position are dealt with seperately, but individual coordinates are not. So if you specify a top (or left) coordinate, you should also specify a left (or top) coordinate, and similar for size.

If you specify both a regular and a default value for the same option the regular value is the only one that takes effect.

参数

url ( string )
options ( optional CreateWindowOptions )
callback ( optional function )

回调函数

如果您指定了 callback 参数，它应该指定一个如下形式的函数： 

function(appwindow created_window) {...};
created_window ( appwindow )

current

chrome.app.window.current()

Returns an AppWindow object for the current script context (ie JavaScript 'window' object). This can also be called on a handle to a script context for another page, for example: otherWindow.chrome.app.window.current().
