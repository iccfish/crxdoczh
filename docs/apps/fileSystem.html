Description: Use the chrome.fileSystem module to create, read, navigate, and write to a sandboxed section of the user's local file system. With this API, packaged apps can read and write to a user-selected location. For example, a text editor app can use the API to read and write local documents.
Availability: Google Chrome 23
Permissions: "fileSystem", "fileSystem.write"

类型

AcceptOption

( object )

AcceptOption 的属性

description ( optional string )
This is the optional text description for this option. If not present, a description will be automatically generated; typically containing an expanded list of valid extensions (e.g. "text/html" may expand to "*.html, *.htm").
mimeTypes ( optional array of string )
Mime-types to accept, e.g. "image/jpeg" or "audio/*". One of mimeTypes or extensions must contain at least one valid element.
extensions ( optional array of string )
Extensions to accept, e.g. "jpg", "gif", "crx".

ChooseEntryOptions

( object )

ChooseEntryOptions 的属性

type ( optional string )
Type of the prompt to show. Valid types are 'openFile', 'openWritableFile' or 'saveFile'.

Both 'openFile' and 'openWritableFile' will prompt the user to open an existing file, with 'openFile' returning a read-only FileEntry on success. 'saveFile' will prompt the user to choose an existing file or a new file, and will return a writable FileEntry. Calls to chooseFile with either 'openWritableFile' or 'saveFile' will fail unless the application has the 'write' permission under 'fileSystem'.

The default is 'openFile'.
suggestedName ( optional string )
The suggested file name that will be presented to the user as the default name to read or write. This is optional.
accepts ( optional array of AcceptOption )
The optional list of accept options for this file opener. Each option will be presented as a unique group to the end-user.
acceptsAllTypes ( optional boolean )
Whether to accept all file types, in addition to the options specified in the accepts argument. The default is true. If the accepts field is unset or contains no valid entries, this will always be reset to true.

getDisplayPath

chrome.fileSystem.getDisplayPath(fileentry fileEntry, function callback)

Get the display path of a FileEntry object. The display path is based on the full path of the file on the local file system, but may be made more readable for display purposes.

参数

fileEntry ( fileentry )
callback ( function )

回调函数

callback 参数应该指定一个如下形式的函数： 

function(string displayPath) {...};
displayPath ( string )

chrome.fileSystem.getWritableEntry(fileentry fileEntry, function callback)

Get a writable FileEntry from another FileEntry. This call will fail if the application does not have the 'write' permission under 'fileSystem'.
