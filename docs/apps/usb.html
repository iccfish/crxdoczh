Description: Use the chrome.usb module to interact with connected USB devices. This API provides access to USB operations from within the context of an app or extension. Using this API, apps and extensions can function as drivers for hardware devices.
Availability: Google Chrome Dev Channel Only
Permissions: "usb"
Learn more: Accessing Hardware Devices

API 参考：chrome.usb

类型

Direction

( enumerated string ["in", "out"] )
Direction, Recipient and RequestType all map to their namesakes within the USB specification.

Recipient

( enumerated string ["device", "interface", "endpoint", "other"] )

RequestType

( enumerated string ["standard", "class", "vendor", "reserved"] )

Device

( object )

Device 的属性

handle ( integer )
vendorId ( integer )
productId ( integer )

ControlTransferInfo

( object )

ControlTransferInfo 的属性

direction ( Direction )
The direction of this transfer.
recipient ( Recipient )
The intended recipient for this transfer.
requestType ( RequestType )
The type of this request.
request ( integer )
value ( integer )
index ( integer )
length ( optional integer )
If this transfer is an input transfer, then this field must be set to indicate the expected data length. If this is an output transfer, then this field is ignored.
data ( optional arraybuffer )
The data payload carried by this transfer. If this is an output tranfer then this field must be set.

GenericTransferInfo

( object )

GenericTransferInfo 的属性

direction ( Direction )
The direction of this transfer.
endpoint ( integer )
length ( optional integer )
If this is an input transfer then this field indicates the size of the input buffer. If this is an output transfer then this field is ignored.
data ( optional arraybuffer )
If this is an output transfer then this field must be populated. Otherwise, it will be ignored.

IsochronousTransferInfo

( object )

IsochronousTransferInfo 的属性

transferInfo ( GenericTransferInfo )
All of the normal transfer parameters are encapsulated in the transferInfo parameters. Note that the data specified in this parameter block is split along packetLength boundaries to form the individual packets of the transfer.
packets ( integer )
The total number of packets in this transfer.
packetLength ( integer )
The length of each of the packets in this transfer.

TransferResultInfo

( object )

TransferResultInfo 的属性

resultCode ( optional integer )
A value of 0 indicates that the transfer was a success. Other values indicate failure.
data ( optional arraybuffer )
If the transfer was an input transfer then this field will contain all of the input data requested.

方法

findDevices

chrome.usb.findDevices(FindDevicesOptions options, function callback)

Finds the first instance of the USB device specified by the vendorId/ productId pair and, if permissions allow, opens it for use. Upon successfully opening a device the callback is invoked with a populated Device object. On failure, the callback is invoked with null.

参数

options ( FindDevicesOptions )
The properties to search for on target devices.
callback ( function )
Invoked with the opened Device on success.

回调函数

callback 参数应该指定一个如下形式的函数： 

function(array of Device device) {...};
device ( array of Device )

closeDevice

chrome.usb.closeDevice(Device device, function callback)

Closes an open device instance. Invoking operations on a device after it has been closed is a safe operation, but causes no action to be taken.

参数

device ( Device )
The device to close.
callback ( optional function )
The callback to invoke once the device is closed.

回调函数

如果您指定了 callback 参数，它应该指定一个如下形式的函数： 

function() {...};
