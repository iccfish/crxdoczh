chrome.experimental.speechInput 模块向 Chrome 扩展程序提供语音识别。这一模块仍然是实验性的。有关使用实验性 API 的信息，请参见 chrome.experimental.* API 页面。

清单文件

您必须在扩展程序的清单文件中声明 "experimental" 权限才能使用语音输入 API。例如： 

{
  "name": "我的扩展程序",
  ...
  "permissions": [
    "experimental"
  ],
  ...
}

怎样开始语音识别

扩展程序必须调用 start() 方法开始语音识别。如果提供了回调函数，一旦录音成功开始就会调用。如果发生错误，则会设置 runtime.lastError。

这一 API 对请求它的第一个扩展程序提供对默认录音设备的独占访问，所以当设备正在由其它扩展程序或网页使用时 start() 的任何调用都会失败，并且会设置 runtime.lastError。如果同一个用户配置文件中的另一个扩展程序正在使用这一 API，则会将消息设置为 requestDenied（请求拒绝），否则取决于具体情况将设置为 recordingDeviceInUse（录音设备正在使用中）或者 unableToStart（无法开始）。

要检查当前录音是否处于活动状态，请调用 isRecording() 方法。请注意它仅仅检查 Chrome 浏览器内的音频录音。

怎样获取语音识别结果

监听 onResult 事件接受语音识别结果。 

var callback = function(result) { ... };

chrome.experimental.speechInput.onResult.addListener(callback);

result 对象包含 experimental.speechInput.SpeechInputResultHypothesis 数组，按照可能性递减的顺序排列。

当结果接收到时录音自动停止，这时可以从结果回调函数中再次调用 start()。

要在语音识别过程中处理错误，请监听 onError 事件。 

var callback = function(error) { ... };

chrome.experimental.speechInput.onError.addListener(callback);

如果发生了错误录音会自动停止，这时也可以从错误回调函数中再次调用 start()。

怎样停止录音

要停止语音识别，请调用 stop() 方法。如果提供了回调函数，一旦录音成功停止就会调用。如果发生错误则会设置 runtime.lastError。

其他功能

  • onSoundStart - 开始检测到声音（之前安静）时产生这一事件。
  • onSoundEnd - 检测到声音停止（持续一段时间的安静）时产生这一事件。

例子

如下例子演示如何通过 JavaScript 消息框显示最有可能的识别结果。

function checkStart() {
  if (chrome.runtime.lastError) {
    alert("无法开始语音输入：" + chrome.runtime.lastError.message);
  }
}

function recognitionFailed(error) {
  alert("语音输入失败：" + error.code);
}

function recognitionSucceeded(result) {
  alert("识别到“" + result.hypotheses[0].utterance + "”，可信度为 " + result.hypotheses[0].confidence);
}

chrome.experimental.speechInput.onError.addListener(recognitionFailed);
chrome.experimental.speechInput.onResult.addListener(recognitionSucceeded);
chrome.experimental.speechInput.start({ "language": "en" }, checkStart);

类型

SpeechInputStartOptions

( object )
描述用于语音识别的选项。

SpeechInputStartOptions 的属性

language ( optional string )
要识别的语言的 BCP-47 语言代码。当设置为 'auto' 或者未定义时，默认为用户首选的内容语言。如果不支持或无效的话则使用 'en-US'。
grammar ( optional string )
要使用的语法名称，默认为 'builtin:dictation'。
filterProfanities ( optional boolean )
启用或禁用脏话过滤功能，如果未设置的话会使用 Chrome 浏览器的默认过滤设置。

SpeechInputError

( object )
描述语音输入错误的对象。

SpeechInputError 的属性

code ( enumerated string ["noSpeechHeard", "noResults", "captureError", "networkError"] )
表示错误情况的代码。

SpeechInputResultHypothesis

( object )
描述语音识别猜测的对象。

SpeechInputResultHypothesis 的属性

utterance ( string )
这一猜测的文本。
confidence ( double )
这一猜测的可信度，从 0（最低）到 1（最高）。

SpeechInputResultEvent

( object )
包含识别结果的对象。

SpeechInputResultEvent 的属性

hypotheses ( optional array of SpeechInputResultHypothesis )
一个数组，含有零个或多个按照可能性递减排列的对象，描述稳定的候选猜测。

方法

start

chrome.experimental.speechInput.start(SpeechInputStartOptions options, function callback)

请求开始录音，作为新的语音识别会话。

参数

options ( optional SpeechInputStartOptions )
用于这一次语音识别的选项。
callback ( optional function )
语音识别会话成功开始录音时调用。

回调函数

如果您指定了 callback 参数，它应该指定一个如下形式的函数： 

function() {...};

stop

chrome.experimental.speechInput.stop(function callback)

请求停止正在进行的语音识别。

参数

callback ( optional function )
当音频录音停止，并且所有未决的识别结果已完成时调用。

回调函数

如果您指定了 callback 参数，它应该指定一个如下形式的函数： 

function() {...};

isRecording

chrome.experimental.speechInput.isRecording(function callback)

确定当前在 Chrome 浏览器中是否有正在进行的音频录音（不仅限于这一 API）。

参数

callback ( optional function )

回调函数

如果您指定了 callback 参数，它应该指定一个如下形式的函数： 

function(boolean result) {...};
result ( boolean )
如果在 Chrome 浏览器中有正在进行的音频录音则设为 true，否则为 false。

事件

onError

chrome.experimental.speechInput.onError.addListener(function(SpeechInputError error) {...});

在语音识别发生错误时调用。

监听器参数

error ( SpeechInputError )
报告的错误。

onSoundStart

chrome.experimental.speechInput.onSoundStart.addListener(function() {...});

当系统开始检测到输入数据中的声音时调用。

onSoundEnd

chrome.experimental.speechInput.onSoundEnd.addListener(function() {...});

当系统检测到足够时间的安静，认为正在进行的语音已经结束时调用。

onResult

chrome.experimental.speechInput.onResult.addListener(function(SpeechInputResultEvent event) {...});

在语音识别结果可用时调用。

监听器参数

event ( SpeechInputResultEvent )
包含语音识别结果的对象。
