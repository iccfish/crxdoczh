该页面列出最近版本中有关API和清单文件的更改。

Google Chrome 22

新的 API

  • 定时器 API 让您安排代码周期性地或者在将来的某一特定时间执行。
  • 运行时 API 包含有用的方法来处理系统范围的事件。

对现有特性的增强

  • 后台页面可以选择不持久运行，使用一种我们称之为事件页面的特性。事件页面仅在使用时运行，并且在空闲时卸载，以节约资源。

Google Chrome 21

新特性

  • 清单文件的 sandbox 项允许您将扩展程序内的某些页面规定为自动运行在低权限的沙箱中，该沙箱不会受到 content_security_policy 的限制。
  • 输入法 API 允许扩展程序在 Chrome OS 上实现输入法。

对现有特性的增强

  • types.ChromeSetting.set 方法现在增加了 regular_only 区域。
  • browsingData.RemovalOptions 现在增加了 originTypes 属性。
  • management.uninstall 方法现在增加了 showConfirmDialog 参数。
  • contextMenus.create 方法现在允许您为每一个项目指定唯一的标识符，这是为了与新的 contextMenus.onClicked 事件搭配使用，来区分单击的项目。
  • browserAction.setIcon 和 pageAction.setIcon 方法现在接受可选的回调函数。
  • privacy.websites 命名空间现在增加了 protectedContentEnabled 属性。
  • tabs.move 方法的 index 参数现在可以接受 -1 值，表示标签页应该放在最后。
  • tabs.highlight 方法的 windowId 参数现在是可选的。

Google Chrome 20

新的 API

  • 存储 API 使您储存、获取和追踪用户数据的更改。

清单文件更改

  • 清单文件版本 1 在 Chrome 18 中弃用，并且会根据清单文件版本1支持计划逐渐减少支持。

对现有 API 的增强

  • chrome.contextMenus 的 contextMenus.create 和 contextMenus.update 方法现在增加了 enabled 参数。
  • 隐私设这 API 的 privacy.services 对象现在增加了 spellingServiceEnabled 设置。
  • chrome.tabs 的 tabs.executeScript 和 tabs.insertCSS 方法现在增加了 runAt 参数。

对现有 API 的更改

  • sendRequest() 方法已弃用，请使用 sendMessage() 方法： runtime.sendMessage 和 tabs.sendMessage。
  • external_extensions.json 文件已弃用，您应该使用单独的 <扩展程序标识符>.json 文件。

Google Chrome 19

新的 API

  • 浏览数据 API 许您从用户的本地配置文件中移除浏览数据。
  • 常用网站 API 使您能够访问显示在“打开新的标签页”页面上的最常访问的网站。

对现有 API 的增强

  • 窗口 API 的 windows.Window 对象现在增加了 alwaysOnTop 属性，并支持"fullscreen"（全屏幕）状态。
  • chrome.tabs 的 tabs.query 方法现在增加了 currentWindow 和 lastFocusedWindow 参数。
  • 浏览器按钮 API 增加了以下这些新的获取函数： browserAction.getTitle、browserAction.getBadgeText、browserAction.getBadgeBackgroundColor 和 browserAction.getPopup。
  • The 页面按钮 API 增加了以下这些新的获取函数： pageAction.getTitle 和 pageAction.getPopup。

Google Chrome 18

新的 API

  • 调试器 API 让您使用 Chrome 浏览器的开发者工具远程访问一个或多个标签页。
  • 开发者工具 APIs 为扩展 Chrome 浏览器的开发者工具提供支持。审查窗口 API 让您与审查的窗口交互，网络 API 让您获取有关网络请求的信息，面板 API 让您添加面板及侧边栏。
  • 页面捕捉 API 让您将标签页保存为 MHTML。
  • 隐私设置 API 让您控制 Chrome 浏览器中会影响用户隐私策略的特性。

清单文件更改

  • 新的清单文件版本字段指定您的扩展操作包需要的清单文件版本。从 Chrome 18 开始，您应该使用清单文件版本 2。
  • 新的内容安全策略（CSP）字段用来定义扩展程序的策略，指定扩展程序能够载入和执行的内容类型。
  • 大部分后台页面仅包括脚本文件的列表，对于这些后台页面，您可以使用新的 background.scripts 属性，Chrome 浏览器会自动为您生成后台页面。
  • 新的 web_accessible_resources 字段指定希望在网页中使用的打包资源的路径。

对现有 API 的增强

  • 窗口 API 现在增加了 WINDOW_ID_CURRENT 属性。
  • chrome.tabs 的 tabs.create 和 tabs.update 方法现在增加了 openerTabId 参数。

Google Chrome 17

新的 API

  • 网络请求 API 让您实时地拦截、阻止或者修改请求，并且可以监控与分析流量。

清单文件更改

  • 内容设置 API、网页导航 API、网络请求 API的权限可以是可选的。

对现有 API 的增强

  • 管理 API的 management.ExtensionInfo 对象现在增加了 disabledReason 属性。

对现有 API 的更改

  • 多功能狂 API 现在可以在分离隐身模式。

Google Chrome 16

新的 API

  • 网页导航 API 使扩展程序接收有关导航请求的详情，您可以使用这一 API 跟踪导航事件。
  • 可选权限 API 让您控制何时向用户展现权限请求。
  • The 内容设置 API 使扩展程序可以针对不同的站点（而不是全局地）自定义 Chrome 浏览器的行为，您可以使用这一 API 控制网站能否使用诸如 Cookie、JavaScript 和插件之类的特性。

清单文件更改

  • 新的 requirements 字段允许您预先声明扩展程序的要求。例如，您可以使用这一字段指定您的应用要求 3D 图形支持，这样就可以使用诸如 CSS 3D 变换或 WebGL 之类的功能。

对现有 API 的增强

  • 新的 tabs.query 方法获取具有指定属性的所有标签页，或者如果没有指定属性的话获取所有标签页。
  • 新的 tabs.reload method 方法重新载入标签页，并包含一个保留重新载入标签页本地缓存的选项。
  • 管理 API 的 management.ExtensionInfo 对象现在增加了 updateURL 属性。
  • 您现在可以通过向 external_extensions.json 添加 supported_locales 属性的方式来限制外部扩展程序支持的语言。

对现有 API 的更改

  • getAllInWindow() 和 getSelected() 方法已弃用。要获得指定窗口中所有标签页的详情，请使用 tabs.query，并传递参数 {'windowId': windowId}。要获得指定窗口中选定的标签页，请使用 chrome.tabs.query()，并传递参数 {'active': true}。
  • 您不再需要为 tabs.update 方法指定 tabId，未提供 tabId时默认为当前窗口的选定标签页。
  • Mac OS 上外部扩展程序文件的所有者现在可以是 wheel 组（或管理员组）内的用户。
  • "panel"（面板）窗口类型不再需要 "experimental" 权限。默认情况下，除非指定了 --enable-panels 命令行参数，"panel" 类型将创建一个弹出窗口。

Google Chrome 15

新的 API

  • 网上应用店 API 使您可以在您的网站中“内嵌”应用与扩展程序的安装功能。

清单文件更改

  • 新的 offline_enabled 字段让您指定您的应用即使在没有网络连接的情况下也能够很好地工作。

对现有 API 的增强

  • 您可以使用新的管理 API 方法 management.getPermissionWarningsById 和 management.getPermissionWarningsByManifest 获取权限警告信息。
  • 管理 API 的 management.ExtensionInfo 对象增加了一个新的属性：offlineEnabled。
  • 现在您可以使用 __MSG_messagename__ 占位符为您的内容脚本 CSS 文件添加国际化支持。

对现有 API 的更改

  • 如果扩展程序没有 "tabs" 权限的话，将会向 tabs.update 方法的回调函数传递 null 而不是标签页详情。

Google Chrome 14

新的 API

  • 文字语音转换 API 使扩展程序可以通过文字产生语音。
  • 文字语音转换引擎 API 使扩展程序实现文字语音转换（TTS）引擎。
  • 通过 Chrome 网上应用店发布的扩展程序现在可以包含 Native Client 模块，而不需要特别的清单文件项。您可以参考 NetHack 例子。有关更多信息，请参见公告（英文）和 Native Client 文档（英文）。

清单文件更改

  • 新的 content_security_policy 字段有助于阻止您的扩展程序中的跨站脚本漏洞。
  • 新的 nacl_modules 字段让您将 Native Client 模块注册为 MIME 类型的内容处理器。

对现有 API 的增强

  • 右键菜单项现在可以出现在 file:// 或 chrome:// 协议的文档中了。以前，它们仅限于 http:// 或 https:// 协议的文档。
  • windows.update 的 updateInfo 中可选的 drawAttention 属性让您指定是否应该提醒用户切换至该窗口。
  • 新的 bookmarks.getSubTree 函数让您获取一部分的书签树。

对现有 API 的更改

  • tabs.remove 和 tabs.onRemoved 不再需要 "tabs" 权限。

Google Chrome 13

新的 API

  • 代理设置 API 让扩展程序管理 Chrome 浏览器的代理设置。
  • types.ChromeSetting 原型提供浏览器设置的接口，由代理设置 API 使用。
  • 仅用于 Chrome OS： 文件浏览器 API 让您扩展Chrome OS的文件浏览器。

清单文件更改

  • "content_scripts" 字段中新增的 exclude_matches 项 让您更精确地定位您的内容脚本。有关更多详情，请参见匹配表达式和范围。
  • 新的 "clipboardRead" 和 "clipboardWrite" 权限指定 document.execCommand() 的能力。

对现有 API 的增强

  • 内容脚本现在可以向所属扩展程序能够访问的相同站点发出跨站 XMLHttpRequests，不用再通过后台页面转发这些请求。
  • 现在您在导入的油猴（Greasemonkey）脚本中可以使用 @run-at 来控制何时插入脚本，它和内容脚本中的 run_at 工作方式相同。

Google Chrome 12

对现有 API 的增强

  • 两个新的 chrome.extension 方法——extension.isAllowedFileSchemeAccess 和 extension.isAllowedIncognitoAccess，让您确定您的扩展程序是否具有额外的访问权限，这些由用户通过扩展程序管理页面（chrome://extensions）。
  • windows.create 方法现在可以传递 focused 值。以前，所有新窗口都具有键盘焦点，现在您可以创建新窗口而不打断用户的输入。
  • 如果清单文件中指定了 "experimental" 权限，您的扩展程序可以在 windows.create 方法或 windows.Window 类型中指定 "panel" 作为 type 属性的值。
  • chrome.cookies 的 cookies.onChanged 事件现在增加了 cause 参数。
  • chrome.contextMenus 的 contextMenus.create 和 contextMenus.update 方法现在允许您指定 contexts 值 "frame"。

Google Chrome 11

对现有 API 的更改

  • 由于安全原因，您将不能在任何标签页上调用 tabs.captureVisibleTab，您必须拥有那一标签页显示的 URL 的主机权限。要保留原来的行为，请指定 <all_urls> 主机权限。

对现有 API 的增强

  • 管理 API 的 management.ExtensionInfo 对象现在增加了 homepageUrl 属性。
  • 管理 API 现在允许您获得已禁用的应用和扩展程序的图标。另外，您现在也可以修改普通图标的 URL 获得禁用时的图标。有关细节请参见 management.IconInfo。
  • Cookie API 的 cookies.set 和 cookies.remove 方法现在可以传递回调函数。

Google Chrome 10

清单文件更改

  • 新的 background 权限扩展了 Chrome 浏览器的生命周期，允许您的扩展程序即使在 Chrome 浏览器没有窗口打开时运行。

对现有 API 的增强

  • windows.create 方法现在增加了 tabId 字段，您可以使用它将标签页或面板移动到新窗口中。

    注： 在以前的发行注记中错误地将这一更改标记为 Chrome 9。

Google Chrome 9

新的 API

  • 多功能框 API 允许您在 Chrome 浏览器的地址栏中注册一个关键字。

清单文件更改

  • homepage_url 属性允许您指定扩展程序或应用的主页。

对现有 API 的增强

  • tabs.Tab 对象现在增加了 pinned 属性，反映在多个 chrome.tabs 方法中。例如，您可以创建一个固定的标签页。
  • windows.create 方法现在可以传递 URL 列表，让您在新窗口中创建多个标签页。
  • 新的 management.get 方法让您获取有关指定扩展程序或应用的信息。

Google Chrome 8

在 Chrome 8 中没有值得一提的 API 或清单文件更改。

Google Chrome 7

新的 API

  • 现在所有用户都能安装应用，打包应用可以使用扩展程序的 API。
  • 管理 API 让您与已安装的应用和扩展程序共同协作。

清单文件更改

  • 引入了分离的隐身模式并作为可安装网上应用的默认值（也可用于扩展程序）。
  • 标签页 API 的 create() 和 update() 方法不再需要 "tabs" 权限，避免了导致吓人对话框的一种常见原因。

Google Chrome 6

新的 API

  • 右键菜单 API 允许您向页面或页面上的指定对象添加右键菜单项。
  • Cookie API 许您管理浏览器的 Cookie 系统。
  • 空闲处理 API 允许您检测计算机空闲状态的更改。

对现有 API 的增强

  • extension.getViews 方法现在可以返回弹出视图。
  • 新增的 windows.WINDOW_ID_NONE 常数表示焦点离开浏览器。
  • 新增的 tabs.getCurrent 方法返回与当前正在执行的脚本对应的标签页。

清单文件更改

  • geolocation 权限让扩展程序访问用户的地理位置。
  • 匹配表达式 现在可以选择所有协议或所有 URL。
  • 访问 file:/// 的 URL 不再引发“访问您的计算机”安全警告，但是现在需要用户手工从扩展程序管理页面允许这一操作。
