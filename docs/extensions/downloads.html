下载 API 允许您以编程方式开始下载以及监视、操纵、搜索下载的文件。

清单文件

您必须在扩展程序的清单文件中声明 "downloads" 权限才能使用该 API，此外对于您可能会传递给 download 的主机还需要主机权限。 

{
  'name': '我的扩展程序',
  ...
  'permissions': [
    'downloads',
    '*://*.google.com'
  ],
  ...
}

如果 URL 的主机名没有在权限中指定，download 调用回调函数时将传递值为 null 的 downloadId，并设置 runtime.lastError 对象，表示扩展程序没有权限访问主机名。

例子

您可以在 examples/api/downloads 目录中找到使用下载模块的简单例子。有关其他例子以及查看源代码的帮助，请参见示例。

类型

HeaderNameValuePair

( object )

HeaderNameValuePair 的属性

name ( string )
HTTP 头信息的名称。
value ( string )
HTTP 头信息的值。

HttpMethod

( enumerated string ["GET", "POST"] )

DownloadOptions

( object )

DownloadOptions 的属性

url ( string )
要下载的 URL。
filename ( optional string )
相对于下载文件夹的路径，包含下载得到的文件。
saveAs ( optional boolean )
使用选择文件对话框，允许用户选择文件名。
method ( optional HttpMethod )
@!--@The HTTP method to use if the URL uses the HTTP[S] protocol.--> 如果 URL 使用 HTTP[S] 协议，则为使用的 HTTP 方法。
headers ( optional array of HeaderNameValuePair )
如果 URL 使用 HTTP[S] 协议，则包含要与请求一起发送的附加 HTTP 头信息。每一条头信息都由词典表示，包含键 name 以及 value 或 binaryValue 中的某一个，仅限于 XMLHttpRequest 中允许的头信息。
body ( optional string )
POST 方法的正文。

DangerType

( enumerated string ["file", "url", "content", "uncommon", "host", "safe"] )
file
下载项的文件名可疑。
url
下载项的 URL 已知是恶意的。
content
已下载的文件已知是恶意的。
uncommon
下载项的 URL 不常见，可能有风险。
host
下载项来自已知发布恶意软件的主机，可能有风险。
safe
下载项对用户的计算机没有已知风险。
这些字符串常数不会更改，但是 DangerType 允许的值可能会更改。

State

( enumerated string ["in_progress", "interrupted", "complete"] )
in_progress
下载项当前正在从服务器接收数据。
interrupted
错误中断了与文件主机之间的连接。
complete
下载成功完成。
这些字符串常数不会更改，但是 State 允许的值可能会更改。

DownloadItem

( object )

DownloadItem 的属性

id ( integer )
在不同浏览器会话间持久存在的标识符。
url ( string )
绝对 URL。
filename ( string )
绝对本地路径。
incognito ( boolean )
如果该下载项记录在历史记录中则为 false，没有记录则为 true。
danger ( DangerType )
表示该下载项是否被识别为安全的或者已知是可疑的。
dangerAccepted ( optional boolean )
如果用户接受这一危险项目的下载则为 true。
mime ( string )
文件的 MIME 类型。
startTime ( string )
下载开始的时间，以 ISO 8601 格式表示，可以直接传递给 Date 的构造函数：chrome.downloads.search({}, function(items){items.forEach(function(item){console.log(new Date(item.startTime))})})。
endTime ( optional string )
下载结束的时间，以 ISO 8601 格式表示，可以直接传递给 Date 的构造函数：chrome.downloads.search({}, function(items){items.forEach(function(item){console.log(new Date(item.endTime))})})。
state ( State )
表示下载项处于正在进行、已中断还是已完成的状态。
paused ( boolean )
如果下载项已停止从主机读取数据，但是连接仍然保持打开状态则为 true。
error ( optional integer )
表示下载项中断原因的整数。
bytesReceived ( integer )
目前为止从主机接收到的字节数，不考虑文件压缩。
totalBytes ( integer )
整个文件的字节数，不考虑压缩，如果未知则为 -1。
fileSize ( integer )
整个文件解压缩后包含的字节数，如果未知则为 -1。
exists ( boolean )
下载的文件是否还存在，该信息可能不是最新的，因为 Chrome 浏览器不会自动检测文件的删除。要检查文件是否存在，请调用 search()。存在性检查完后，如果文件已删除，会产生 onChanged 事件。注意 search() 不会等到存在性检查完成后再返回，所以 search() 的结果不一定能准确地反映文件系统的情况。此外，尽管可以频繁调用 search()，但是检查文件存在性的频率不会高于每 10 秒一次。

GetFileIconOptions 的属性

图标大小。返回图标的大小为 size×size 像素，图标的默认大小为 32×32 像素。有效的值包括 16 和 32。

方法

download

chrome.downloads.download(DownloadOptions options, function callback)

下载某个 URL。如果该 URL 使用 HTTP[S] 协议，请求会包含当前为主机名设置的所有 Cookie。如果同时指定了 filename 与 saveAs 属性，则会显示另存为对话框，并且初始文件名为指定的 filename。如果下载成功开始，将调用 callback，并传递新的 DownloadItem（下载项目）的 downloadId。如果开始下载时发生了错误，将调用 callback，并传递 downloadId: undefined，并且 runtime.lastError 包含描述性文字。错误字符串并不能保证在不同版本之间保持向后的兼容性，您决不能分析它。

参数

options ( DownloadOptions )
下载的内容与方式。
callback ( optional function )
调用时传递新的 DownloadItem 的标识符。

回调函数

如果您指定了 callback 参数，它应该指定一个如下形式的函数： 

function(integer downloadId) {...};
downloadId ( integer )

search

chrome.downloads.search(DownloadQuery query, function callback)

寻找 DownloadItem（下载项）。将 query 设置为空对象获取所有下载项。要获得某个特定的下载项，只要设置 id 字段。

参数

query ( DownloadQuery )
callback ( function )

回调函数

callback 参数应该指定一个如下形式的函数： 

function(array of DownloadItem results) {...};
results ( array of DownloadItem )

pause

chrome.downloads.pause(integer downloadId, function callback)

暂停下载。如果请求成功，下载项将进入暂停状态。否则，runtime.lastError 将包含错误消息。如果下载项不处于活动状态，该调用会失败。

参数

downloadId ( integer )
要暂停的下载项标识符。
callback ( optional function )
暂停请求完成时调用。

回调函数

如果您指定了 callback 参数，它应该指定一个如下形式的函数： 

function() {...};

resume

chrome.downloads.resume(integer downloadId, function callback)

恢复已暂停的下载。如果请求成功，下载项将继续进行，不再处于暂停状态。否则，runtime.lastError 将包含错误消息。如果下载项不处于活动状态，该调用会失败。

参数

downloadId ( integer )
要恢复的下载项标识符。
callback ( optional function )
恢复请求完成后调用。

回调函数

如果您指定了 callback 参数，它应该指定一个如下形式的函数： 

function() {...};

cancel

chrome.downloads.cancel(integer downloadId, function callback)

取消下载。回调函数运行时，下载已取消、已完成、已中断或者不再存在。

参数

downloadId ( integer )
要取消的下载项标识符。
callback ( optional function )
取消请求完成后调用。

回调函数

如果您指定了 callback 参数，它应该指定一个如下形式的函数： 

function() {...};

getFileIcon

chrome.downloads.getFileIcon(integer downloadId, GetFileIconOptions options, function callback)

获取指定下载项的图标。对于新的下载项，文件图标将在 onCreated 事件收到后才可用。在下载进行时该函数返回的图像可能与下载完成后返回的图像不同。获取图标的方式取决于平台，通过查询下层操作系统或工具包的方式进行。因此，返回的图标取决于各种因素，包括下载状态、平台、已注册的文件类型以及视觉主题。如果无法确定文件图标，runtime.lastError 将会包含错误消息。

参数

downloadId ( integer )
下载项的标识符。
options ( optional GetFileIconOptions )
callback ( function )
调用时传递代表下载项的图标。

回调函数

callback 参数应该指定一个如下形式的函数： 

function(string iconURL) {...};
iconURL ( optional string )

open

chrome.downloads.open(integer downloadId)

如果 DownloadItem（下载项）已完成则立即打开下载的文件，否则通过 runtime.lastError 返回错误。项目第一次打开时将产生 onChanged 事件。

参数

downloadId ( integer )
已下载文件的标识符。

show

chrome.downloads.show(integer downloadId)

在文件管理器中，打开已下载文件所在的文件夹并显示它。

参数

downloadId ( integer )
已下载文件的标识符。

erase

chrome.downloads.erase(DownloadQuery query, function callback)

从历史记录中删除匹配的 DownloadItem（下载项），匹配 query 的每一个 DownloadItem 都会产生一次 onErased 事件，然后调用 callback。

参数

query ( DownloadQuery )
callback ( optional function )

回调函数

如果您指定了 callback 参数，它应该指定一个如下形式的函数： 

function(array of integer erasedIds) {...};
erasedIds ( array of integer )

acceptDanger

chrome.downloads.acceptDanger(integer downloadId)

提示用户接受有风险的下载，不会自动接受有风险的下载。如果下载项被接受，将产生 onChanged 事件，否则什么都不会发生。当所有数据都获取至临时文件后，如果下载项没有风险或者已被接受，该临时文件将被重命名为目标文件名，state 更改为 'complete'，并产生 onChanged 事件。

参数

downloadId ( integer )
DownloadItem 的标识符。

drag

chrome.downloads.drag(integer downloadId)

开始将已下载的文件拖动到另一个应用程序。

参数

downloadId ( integer )

事件

onCreated

chrome.downloads.onCreated.addListener(function(DownloadItem downloadItem) {...});

下载开始时产生该事件，并传递 DownloadItem 对象。

监听器参数

downloadItem ( DownloadItem )

onErased

chrome.downloads.onErased.addListener(function(integer downloadId) {...});

当下载项从历史记录中删除时产生，并传递 downloadId。

监听器参数

downloadId ( integer )
已删除的 DownloadItem 的 id。

onChanged

chrome.downloads.onChanged.addListener(function(DownloadDelta downloadDelta) {...});

当某个 DownloadItem 除 bytesReceived 以外的任何属性更改时，将会产生该事件，并传递 id 以及描述属性更改的对象。

监听器参数

downloadDelta ( DownloadDelta )
